I2C、SPI、UART 简介

发表于2026-01-20|更新于2026-01-20|通信协议
|浏览量:

本文为 parlezvoustech 网站中博文 的中文翻译

在嵌入式系统与电子设计中，I2C、SPI 与 UART 是最常见、最基础的三种串行通信协议。它们在速度、复杂度、连线方式以及应用场景上各有侧重。

本文将系统性地介绍三种协议的：

  • 基本原理
  • 通信流程
  • 优缺点
  • 典型应用场景
  • 对比与选型建议

一、I2C 协议（Inter-Integrated Circuit）

1. 基本原理

I²C 是一种 同步、双向、半双工 的串行通信协议，支持 多主多从架构。

其主要特点如下：

  • 仅使用 两条信号线
    • SDA（Serial Data）：串行数据线
    • SCL（Serial Clock）：串行时钟线
  • 所有通信 均由主设备发起
  • 主设备之间、从设备之间 不能直接通信
  • 通过 设备地址 区分不同从设备

2. I2C 的读写流程

I2C 通信分为 读操作（Slave → Master） 和 写操作（Master → Slave），二者流程相似但方向不同。

2.1 读操作（Slave → Master）

  1. START 条件
    主设备在 SCL 为高电平时，将 SDA 拉低，表示通信开始。

  2. 从设备地址 + 读位（R=1）
    主设备发送 7 位从地址，随后发送读控制位。

  3. 应答信号（ACK / NACK）
    从设备若准备就绪，则拉低 SDA 发送 ACK。

  4. 从设备发送数据
    从设备按字节向主设备发送数据。

  5. 主设备应答

    • ACK：继续接收数据
    • NACK：表示不再接收

  6. STOP 条件
    主设备在 SCL 为高电平时，将 SDA 从低拉高，通信结束。

2.2 写操作（Master → Slave）

  1. START 条件
  2. 从设备地址 + 写位（W=0）
  3. 从设备应答 ACK
  4. 主设备发送数据
  5. 从设备逐字节 ACK
  6. STOP 或 RESTART 条件

3. I2C 的优缺点与应用

✅ 优点

  • 仅需 2 根信号线，引脚占用极低
  • 支持多从设备
  • 硬件与软件实现相对简单

❌ 缺点

  • 传输速率较低（相比 SPI）
  • 抗干扰能力有限，不适合长距离
  • 从设备数量增加后，管理复杂度上升

📌 典型应用

  • 传感器（温度、压力、IMU）
  • RTC（实时时钟）
  • LCD / OLED 显示屏
  • 电池管理、LED 控制器

二、SPI 协议（Serial Peripheral Interface）

1. 基本原理

SPI 是一种 高速、同步、全双工 串行通信协议，以速度和效率著称。

SPI 通常使用 4 根信号线：

  • MOSI（Master Out Slave In）
  • MISO（Master In Slave Out）
  • SCK（Serial Clock）
  • SS / CS（Slave Select / Chip Select）

2. SPI 通信流程

  1. 时钟产生
    主设备产生 SCK 时钟信号。

  2. 从设备选择
    主设备将对应 CS 拉低，选中目标从设备。

  3. 数据交换

    • 主设备通过 MOSI 发送数据
    • 从设备通过 MISO 同时返回数据

  4. 全双工通信
    只要 CS 保持有效，双方即可持续高速交换数据。

3. SPI 的优缺点与应用

✅ 优点

  • 极高的数据传输速率
  • 全双工通信
  • 协议简单、时序清晰

❌ 缺点

  • 引脚占用多
  • 多从设备场景下 CS 管理复杂
  • 高速下对布线和 EMI 要求高

📌 典型应用

  • TFT / LCD 显示屏
  • SD / Flash 存储器
  • 无线通信模块（NRF、ESP）
  • 高速 ADC / DAC

三、UART 协议（Universal Asynchronous Receiver / Transmitter）

1. 基本原理

UART 是一种 异步串行通信协议，无需共享时钟，仅依赖双方约定的 波特率。

仅使用两根信号线：

  • TX：发送
  • RX：接收

2. UART 数据帧结构

典型 UART 帧包含：

  • 起始位（Start Bit）
  • 数据位（5~9 位，LSB 先发）
  • 校验位（可选）
  • 停止位（1 或 2 位）

3. UART 工作过程

  1. 空闲时 TX 保持高电平
  2. 起始位拉低，接收端开始采样
  3. 按波特率读取数据位
  4. 校验位用于简单错误检测
  5. 停止位结束本帧数据

常见波特率：9600、19200、38400、57600、115200
双方误差一般需小于 ±10%

4. UART 的优缺点与应用

✅ 优点

  • 硬件简单，仅需 2 根线
  • 无需时钟，系统灵活
  • 支持校验位

❌ 缺点

  • 不适合多设备总线结构
  • 传输速率有限
  • 对波特率一致性敏感

📌 典型应用

  • MCU 与 PC 串口通信
  • GPS 模块
  • 工业设备调试接口
  • RS-232 / RS-485 通信系统

四、I2C / SPI / UART 对比分析

特性 I2C SPI UART
速度 中等（≤ 几 Mbps） 很高（几十 Mbps） 中等
引脚数量 2 ≥4 2
通信方式 半双工 全双工 全双工
多设备支持 优秀 一般 差
距离 短 短 可较长

五、协议选型建议

  1. 追求高速 → SPI
  2. 多设备、少引脚 → I2C
  3. 远距离、调试、简单通信 → UART
  4. 需要全双工 → SPI / UART
  5. 资源受限 MCU → I2C / UART

结语

I2C、SPI 与 UART 各自代表了 不同设计取舍下的通信哲学：

  • I2C：省引脚、结构清晰，适合系统级互联
  • SPI：速度至上，适合高性能数据传输
  • UART：简单可靠，是调试与远距离通信的常青树

在嵌入式系统中，真正的能力并不在于“会不会用”，而在于 是否能在合适的场景选用合适的协议。

建议在项目中多做实验与对比，这将极大提升你对系统架构与通信设计的理解。

🚀 通信，是一切系统协同的基础。
