无锁环形队列

什么是无锁环形队列

无锁环形队列（Lock-Free Ring Queue）是一种用于在并发环境下传输数据的先进先出（FIFO）数据结构。它通过原子操作或内存屏障来保证线程安全，而不是使用传统的互斥锁（Mutex），从而避免了锁竞争带来的性能开销和上下文切换。

SPSC 模型定义

SPSC 是 Single Producer Single Consumer（单生产者单消费者）的缩写。

  • 单生产者 (Producer)：只有一个线程（或中断服务程序）负责向队列中写入数据。
  • 单消费者 (Consumer)：只有一个线程（或中断服务程序）负责从队列中读取数据。

在 SPSC 模型下，实现无锁队列是最简单的，因为不存在多个生产者同时竞争写入位置，也不存在多个消费者同时竞争读取位置的情况。

[!IMPORTANT]

  1. 执行体隔离：生产与消费者不共享执行状态
  2. 数据流单向性：数据仅从生产者流向消费者
  3. 无竞争访问：头尾指针分别由生产者和消费者独占修改

特征 传统锁队列 CAS无锁队列 SPSC无锁队列

核心原理

数据结构

通常基于一个固定大小的数组实现环形缓冲区（Ring Buffer）。主要包含三个要素：

  1. Buffer: 存储数据的数组。
  2. Head (Write Index): 生产者写入数据的索引。
  3. Tail (Read Index): 消费者读取数据的索引。

空与满的判断

为了区分队列“空”和“满”的状态，通常会保留一个存储单元不存数据，或者使用额外的标记。最常用的方法是保留一个空位：

  • 初始化: head = 0, tail = 0
  • 判空 (Empty): head == tail
  • 判满 (Full): (head + 1) % Capacity == tail

无锁机制

在 SPSC 场景下：

  • Head 仅由生产者修改，消费者只读。
  • Tail 仅由消费者修改，生产者只读。

由于没有两个线程同时修改同一个变量，只要保证内存操作的顺序性（Memory Ordering），就不需要加锁。

在现代多核 CPU 上，需要使用内存屏障（Memory Barrier）或 C++11 的 std::atomic 来防止指令重排和确保内存可见性。

  • 生产者：写入数据 -> StoreStore 屏障 -> 更新 head。确保数据在 head 更新前对消费者可见。
  • 消费者：读取数据 -> LoadLoad 屏障 -> 更新 tail。确保读取的是有效数据。

优缺点

优点

  • 高性能：消除了锁竞争，特别是在多核系统上。
  • 低延迟：没有锁的获取和释放开销。
  • 确定性：适合实时系统，避免了优先级反转等问题。

缺点

  • 容量限制：通常是固定大小的数组。
  • 适用性有限：SPSC 仅适用于单生产者单消费者场景，多生产者或多消费者需要更复杂的实现（如 CAS 操作）。
